MariaDB функция ADDDATE

В этом учебном пособии вы узнаете, как использовать MariaDB функцию ADDDATE с синтаксисом и примерами.

Описание

MariaDB функция ADDDATE возвращает дату, после которой был добавлен определенный интервал даты/времени.

Синтаксис

Синтаксис функции ADDDATE в MariaDB:

ADDDATE( date, INTERVAL value unit )

Или

ADDDATE( date, days )

Параметры или аргументы

date
Дата, к которой следует добавить интервал.
days
Количество дней, добавляемых к date (второй синтакс).
value
Значение интервала даты/времени, который вы хотите добавить. Вы можете указать положительные и отрицательные значения для этого параметра (первый синтаксис).
unit
Тип единицы измерения интервала, такой как DAY, MONTH, MINUTE, HOUR и т.д. Это может быть одно из следующих (первый синтаксис):

Примечание

  • Если вы укажете интервал value, который слишком мал для указанного вами unit, функция ADDDATE будет считать, что самая левая часть value интервала не была предоставлена.
  • Функция ADDDATE (только первый синтаксис) является синонимом для функции DATE_ADD.
  • Использование функции ADDDATE с отрицательным значением value в качестве параметра эквивалентно использованию функции SUBDATE.
  • См. также функции: DATE_ADD, DATE_SUB, SUBDATE, ADDTIME и SUBTIME.

Применение

Функцию ADDDATE можно использовать в следующих версиях MariaDB:

  • MariaDB 10

SELECT ADDDATE('2019-05-17 08:44:21.000001', INTERVAL 4 MICROSECOND);
 
--Результат: '2019-05-17 08:44:21.000005'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL -20 SECOND);
 
--Результат: '2019-05-17 08:44:01'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL 25 MINUTE);
 
--Результат: '2019-05-17 09:09:21'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL -2 HOUR);
 
--Результат: '2019-05-17 06:44:21'
 
SELECT ADDDATE('2019-05-17', INTERVAL 10 DAY);
 
--Результат: '2019-05-27'
 
SELECT ADDDATE('2019-05-17', 10);
 
--Результат: '2019-05-27'
 
SELECT ADDDATE('2019-05-17', INTERVAL 12 WEEK);
 
--Результат: '2019-08-09'
 
SELECT ADDDATE('2019-05-17', INTERVAL -3 MONTH);
 
--Результат: '2019-02-17'
 
SELECT ADDDATE('2019-05-17', INTERVAL 3 QUARTER);
 
--Результат: '2020-02-17'
 
SELECT ADDDATE('2019-05-17', INTERVAL 5 YEAR);
 
--Результат: '2024-05-17'
 
SELECT ADDDATE('2019-05-17 08:44:21.000001', INTERVAL '12.000001' SECOND_MICROSECOND);
 
--Результат: '2019-05-17 08:44:33.000002'
 
SELECT ADDDATE('2019-05-17 08:44:21.000001', INTERVAL '3:12.000001' MINUTE_MICROSECOND);
 
--Результат: '2019-05-17 08:47:33.000002'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL '3:12' MINUTE_SECOND);
 
--Результат: '2019-05-17 08:47:33'
 
SELECT ADDDATE('2019-05-17 08:44:21.000001', INTERVAL '1:03:12.000001' HOUR_MICROSECOND);
 
--Результат: '2019-05-17 09:47:33.000002'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL '1:03:12' HOUR_SECOND);
 
--Результат: '2019-05-17 09:47:33'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL '1:03' HOUR_MINUTE);
 
--Результат: '2019-05-17 09:47:21'
 
SELECT ADDDATE('2019-05-17 08:44:21.000001', INTERVAL '7 1:03:12.000001' DAY_MICROSECOND);
 
--Результат: '2019-05-24 09:47:33.000002'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL '7 1:03:12' DAY_SECOND);
 
--Результат: '2019-05-24 09:47:33'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL '7 1:03' DAY_MINUTE);
 
--Результат: '2019-05-24 09:47:21'
 
SELECT ADDDATE('2019-05-17 08:44:21', INTERVAL '7 1' DAY_HOUR);
 
--Результат: '2019-05-24 09:44:21'
 
SELECT ADDDATE('2019-05-17', INTERVAL '5-3' YEAR_MONTH);
 
--Результат: '2024-08-17'
