Представление (VIEW) SQLite

В этом учебном пособии вы узнаете, как создать, изменить, и удалить VIEWS в SQLite с синтаксисом и примерами.

Описание

В SQLite VIEW - это не физическая таблица, а виртуальная таблица, созданная запросом, соединяющим одну или несколько таблиц.

Создать VIEW

Синтаксис

Синтаксис для оператора CREATE VIEW в SQLite:

CREATE VIEW view_name AS
SELECT columns
FROM tables
[WHERE conditions];

view_name
Имя VIEW, который вы хотите создать в SQLite.

WHERE conditions
Необязательный. Условия, которые должны быть выполнены, чтобы записи были включены в VIEW.

Пример

Вот пример того, как использовать оператор CREATE VIEW для создания представления в SQLite:

PgSQL
1
2
3
4
CREATE VIEW active_employees AS
  SELECT employee_id, last_name, first_name
  FROM employees
  WHERE hire_date IS NOT NULL;

В этом примере CREATE VIEW будет создана виртуальная таблица на основе результирующего набора оператора SELECT. Теперь вы можете запросить SQLite VIEW следующим образом:

PgSQL
1
2
SELECT *
FROM active_employees;

Удалить VIEW

Как только VIEW был создан в SQLite, вам может понадобиться удалить VIEW с помощью оператора DROP VIEW.

Синтаксис

Синтаксис для оператора DROP VIEW в SQLite:

DROP VIEW [IF EXISTS] view_name;

view_name
Название представления, которое вы хотите удалить.

WHERE conditions
Необязательный. Если вы не укажете это предложение, а VIEW не существует, оператор DROP VIEW вернет ошибку.

Пример

Вот пример того, как использовать оператор DROP VIEW в SQLite:

PgSQL
1
DROP VIEW active_employees;

Этот пример DROP VIEW удалит SQLite VIEW с именем active_employees.
