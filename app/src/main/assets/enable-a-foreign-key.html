Enable a foreign key

В этом учебном материале вы узнаете, как использовать enable a foreign key (включить внешний ключ) в Oracle/PLSQL с синтаксисом и примерами.

Описание

Вы можете столкнуться с foreign key, который был отключен. Вы можете включить внешний ключ с помощью оператора ALTER TABLE.

Синтаксис

Синтаксис для включения внешнего ключа в Oracle/PLSQL:

ALTER TABLE table_name
ENABLE CONSTRAINT constraint_name;

Если вы создали foreign key следующим образом:

CREATE TABLE supplier
( supplier_id numeric(10) not null,
  supplier_name varchar2(50) not null,
  contact_name varchar2(50),
  CONSTRAINT supplier_pk PRIMARY KEY (supplier_id)
);
 
CREATE TABLE products
( product_id numeric(10) not null,
  supplier_id numeric(10) not null,
  CONSTRAINT fk_supplier
    FOREIGN KEY (supplier_id)
    REFERENCES supplier(supplier_id)
);

В этом примере мы создали primary key (первичный ключ) таблицы supplier под названием supplier_pk. Он состоит только из одного поля supplier_id. Затем мы создали foreign key с именем fk_supplier в таблице products, который ссылается на таблицу supplier, основанный на поле supplier_id.

Если primary key был отключен, и мы хотели бы его включить, то необходимо выполнить следующую команду:

Oracle PL/SQL
1
2
ALTER TABLE products
ENABLE CONSTRAINT fk_supplier;
